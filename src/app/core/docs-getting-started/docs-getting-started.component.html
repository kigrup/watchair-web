Getting started

To start using WatChair you first need to connect to a server where your data can be stored and processed.

You can either host your own instance or connect to one.

Hosting your own server

The server for this project is built into a Docker image. To be able to host your own you must:
  • Have a Docker engine installed in your system. (Docker Desktop is available for Windows, Linux and Mac)
  • Download the latest WatChair server image from Docker Hub:
    • Either from the browser.
    • Or via command line (Docker installation required):
      docker pull raulplesa/watchair-server:latest
Once you have the image you can run it with the following command:
docker run -d -p 42525:42525 --name watchair-server raulplesa/watchair-server:latest

This runs the latest version of the image in your system named "raulplesa/watchair-server", exposing container port 42525 to the local machine port 42525. The running container name is set to "watchair-server" (needed if you want to refer to it via CLI). The -d option means detached, making the container run in the background.

Connecting to a server

Once you have access to a running server, you can set up the application to connect to it. To do this you must go to the Settings section of the web application and set the field Instance URL to the desired IP and port. Now whenever you try to login you will connect to that server.
